Data Table Design Rules

The 5 Rules

1

Every row is one item

2

3

First column = unique ID

4

Use clear column names (TaskName not TN)

5

Start with 5+ sample rows

Think of It Like This

Your Google Sheet = the brain of your app

The columns = types of information

The rows = actual data entries

Like a well-organized notebook — if the notebook is messy, you can't find anything.

LASU 2001 Engineers • Build Without Coding STEP 4 • TEST

Testing Checklist

"Do 5 test actions: create, edit, filter, mark complete, re-open app."
  • CREATE — Add a new task. Does it appear?
  • EDIT — Change a priority. Does it save?
  • FILTER — Switch views. Correct data?
  • COMPLETE — Tap "Mark as Done". Status changes?
  • RE-OPEN — Close and reopen. Data persists?

Common Fixes

  • Won't sync? Check internet, hit Sync button
  • Wrong data? Check column types in Data settings
  • Missing fields? Set columns to "Show" in form view
  • Action broken? Column name must match exactly

LASU 2001 Engineers • Build Without Coding TIPS

Common Mistakes & Fixes

Mistake

  • Trying to build everything at once
  • Spending too long on design before it works
  • Messy column names (spaces, symbols)
  • No sample data in the sheet
  • Not testing each feature as you add it

Fix

  • Build the simplest version first (MVP)
  • Make it work, then make it pretty
  • Use CamelCase: TaskName, DueDate
  • Always add 5+ rows of real-looking data
  • Test one thing at a time

The Art of Talking to AI

Better prompts = better results. Same formula as this morning!

Bad Prompt

"Make me an app"

Too vague. AI doesn't know what, who, or how.

Good Prompt

"Build me a Nigerian recipe app as a single HTML file. Show 6 recipes with names, cooking time, ingredients, and step-by-step instructions. Include search and category filters. Use warm orange colors, mobile-friendly."

Clear, specific, describes features and design.
