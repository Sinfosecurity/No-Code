LASU 2001 Engineers • Build Without Coding STEP 2 • DATA

Data Table Design Rules

"Your app is only as good as your data structure. Let's build the table first."

The 5 Rules

1

Every row is one item (one task, one order, one event)

2

Every column is one piece of info (name, date, status)

3

First column = unique ID (so the app can identify each item)

4

Use clear column names (TaskName not TN)

5

Start with sample data (at least 5 rows)

Think of It Like This

Your Google Sheet is the brain of your app.

The columns are the types of information.

The rows are the actual data entries.

Just like a well-organized notebook — if the notebook is messy, you can't find anything. If the data is messy, the app won't work well.

LASU 2001 Engineers • Build Without Coding STEP 4 • TEST 1:30 - 1:45

Testing Checklist

"Do 5 test actions: create, edit, filter, mark complete, re-open app."

The 5-Point Test

  • CREATE — Add a new task using the form. Does it appear in the list?
  • EDIT — Change a task's priority or category. Does it save?
  • FILTER — Try the different views. Do they show the right tasks?
  • MARK COMPLETE — Tap "Mark as Done". Does the status change?
  • RE-OPEN — Close and reopen the app. Is your data still there?

Common Issues & Fixes

  • App won't sync?
    Check your internet connection and try "Sync" button
  • Wrong data showing?
    Check column types in AppSheet Data settings
  • Form missing fields?
    Make sure all columns are set to "Show" in the form view
  • Action not working?
    Check that the column name matches exactly

LASU 2001 Engineers • Build Without Coding TIPS

Common Mistakes & Fixes

Mistake

  • Trying to build everything at once
  • Spending too long on design before it works
  • Messy column names (spaces, special characters)
  • No sample data in the sheet
  • Ignoring the mobile preview
  • Not testing each feature as you add it

Fix

  • Build the simplest version first (MVP)
  • Make it work, then make it pretty
  • Use CamelCase: TaskName, DueDate
  • Always add 5+ rows of real-looking data
  • Check mobile view after every change
  • Test one thing at a time — create, edit, delete

Remember: Your win is not the final design. Your win is the working prototype.
