Lab 2: Build Your Data Table 0:30 - 0:45

Step 1: Open Google Sheets

1 Go to sheets.google.com

2 Click "+ Blank spreadsheet"

3 Name it: "Smart Task Tracker" (click "Untitled spreadsheet" at top-left)

Step 2: Create Column Headers

In Row 1, type these headers (one per cell, A1 through H1):

Use CamelCase for column names (TaskName, DueDate) — no spaces. This makes AppSheet work better.
Date format issues? Use the format YYYY-MM-DD (e.g., 2026-03-01). Or use Google Sheets date formatting: select the column → Format → Number → Date.

Lab 3: Generate Your App in AppSheet 0:45 - 1:10

Step 1: Open AppSheet

1 Go to appsheet.com

2 Click "Sign in" and use your Google account

3 Click "Create" → "App" → "Start with existing data"

4 Give your app a name: "Smart Task Tracker"

5 Select "Google Sheets" as your data source

6 Find and select your "Smart Task Tracker" spreadsheet

7 Click "Customize with AppSheet"

Your app is now generated!

AppSheet automatically created a basic app from your spreadsheet. You should see a list of your tasks in the preview panel on the right.

Step 2: Configure Column Types

Go to Data → Columns in the left panel and set these types:

ColumnTypeSettings
IDNumberCheck "Is a key" (unique identifier)
TaskNameTextRequired: Yes
CategoryEnumValues: School, Work, Personal, Business
PriorityEnumValues: Low, Medium, High
DueDateDate(auto-detected)
StatusEnumValues: To Do, In Progress, Done
OwnerText(default)
NotesLongText(default)

Step 3: Create Views

Go to UX → Views in the left panel:

  1. All Tasks — View type: Table or Deck. Data: your sheet. This is the default view.
  2. Add New View: "Calendar" — View type: Calendar. Set the date field to DueDate.
  3. Add New View: "By Category" — View type: Deck. Group by: Category.
Click the phone preview on the right side to see how your app looks on a mobile device. Check it after every change!

Lab 4: Improve UX and Logic 1:10 - 1:30

Add Format Rules (Color Coding)

Go to UX → Format Rules:

  1. Click "+ New Format Rule"
  2. Name it: "High Priority Red"
  3. Set condition: [Priority] = "High"
  4. Set highlight color to Red
  5. Click Save

Repeat for:

  • Medium Priority → Yellow highlight — Condition: [Priority] = "Medium"
  • Done Tasks → Gray/muted — Condition: [Status] = "Done"

Add Action Button: "Mark as Done"

  1. Go to Actions (in the left panel)
  2. Click "+ New Action"
  3. Name: "Mark as Done"
  4. For table: your data table
  5. Do this: "Set the values of some columns in this row"
  6. Set column: Status = "Done"
  7. Appearance: Choose an icon (checkmark works great)
  8. Position: "Display prominently"
  9. Click Save

Test your action!

In the app preview, open any task and look for the "Mark as Done" button. Tap it — the status should change to "Done" and the row should turn gray (if you set the format rule).

Lab 5: Test and Validate 1:30 - 1:45

The 5-Point Test Checklist

Do each of these actions in your app and confirm they work:

  • CREATE: Tap the "+" button to add a new task. Fill in all fields. Does it appear in your list?
  • EDIT: Open an existing task. Change the priority from "Low" to "High". Does it save?
  • FILTER: Switch to your Calendar view. Do tasks appear on the correct dates?
  • MARK COMPLETE: Open a task and tap "Mark as Done". Does the status change?
  • RE-OPEN: Close the app preview and reopen it. Is all your data still there?
New task not appearing? Click the Sync button (circular arrow) at the top of the app preview. AppSheet syncs data with your Google Sheet.
Action button not showing? Check that you set "Display prominently" in the action settings. Also make sure the action is assigned to the correct table.
Wrong data types? Go back to Data → Columns and make sure Category, Priority, and Status are set to "Enum" type with the correct values.
